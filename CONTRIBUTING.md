# Contributing to Web Calculator

Thanks for your interest in contributing to this project! All contributions are welcome — whether it's a bug fix, improvement, new feature, or documentation update. Please follow the guidelines below to ensure a smooth process.

## How to Contribute

1. Fork the repository.  
2. Create a new branch for your change:  
   `git checkout -b feature/my-change`  
3. Make your changes in the code.  
4. Test your changes to make sure everything works correctly.  
5. Commit your work following the commit guidelines below.  
6. Push your branch:  
   `git push origin feature/my-change`  
7. Open a Pull Request.

## Commit Guidelines

- Use **present tense**: "Add feature", not "Added feature"  
- Keep the **title under 50 characters**  
- Capitalize the first letter  
- Do **not** put a period at the end of the title  
- Leave an empty line before writing a detailed description (optional)  

## Pull Requests

When submitting a Pull Request:

- Provide a short explanation of what you changed and why  
- Attach screenshots if your change affects the UI  
- Make sure your branch is up to date with `main` before submitting:  
  `git pull origin main`

## Code Style

Please follow these simple rules:

- Keep code clean and readable  
- Use meaningful variable and function names  
- Avoid unnecessary duplication  
- Format HTML, CSS, and JS properly

## Behaviour

Be respectful and friendly with other contributors. Constructive feedback is always welcome — toxicity is not allowed.

Thanks again for contributing! Together, we can make this project better.
